## 📁 Файловая система и команды управления файлами/каталогами
Добавлена собственная файловая система с поддержкой следующих команд:

- `mkdir="имя_каталога_или_путь"` — создать каталог.
- `rmdir="имя_каталога_или_путь"` — удалить каталог.
- `mkfile="имя_файла_или_путь"` — создать файл.
- `rmfile="имя_файла_или_путь"` — удалить файл.
- `write="путь_к_файлу"->"текст"` — записать текст в файл.
- `read="путь_к_файлу"` — прочитать содержимое файла.
- `copy="путь_источника"->"путь_назначения"` — скопировать файл/каталог.
- `move="путь_источника"->"путь_назначения"` — переместить файл/каталог.
- `cd="путь"` — перейти в указанный каталог.
- `cd..` — перейти на уровень выше.
- `pwd` — отобразить текущий путь.
- `ls` — отобразить содержимое текущего каталога.
- `ls="путь"` — отобразить содержимое указанного каталога.

**Дополнительно:**  
Поддержка множественного выполнения через символ `|`, например:  
`mkdir=docs|images|bin` — создаст сразу три каталога.

## 📃 Логирование
- ✅ **InitSystem** больше не пишет логи в консоль — теперь они сохраняются в `Root/Logs/Initialization_System.log`.
- ✅ **Ядро** пишет логи в файл `Root/Logs/Kernel.log`.
- 📂 Автоматически создаётся директория `Root/Logs` при необходимости.

---

## 🔍 Модуль просмотра логов
Поддерживает просмотр с пагинацией и цветовую подсветку по типу логов:

- `logs-kernel` — просмотр логов ядра.
- `logs-initialization_system` — просмотр логов инициализации.
- `logs-kernel-clear` — очистка логов ядра.
- `logs-initialization_system-clear` — очистка логов инициализации.
- `logs-clear` — полная очистка всех логов.

## ➕ Множественное выполнение команд
- Используйте `&&` для последовательного выполнения нескольких команд.  
  Пример:  
  `mkdir=demo&&cd=demo` — создаёт каталог `demo` и переходит в него.
